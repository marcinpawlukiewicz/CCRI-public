### Points system (0–6 points total)

You will compute three point variables:

- `digitPoints` (0–2)
- `capitalPoints` (0–2)
- `symbolsPoints` (0–2)

#### Digits (0–2 points)

- Must contain at least **1 digit** (`0–9`) to meet requirements
- Scoring:
  - 0 digits → **0 points**
  - 1 digit → **1 point**
  - 2+ digits → **2 points**

#### Uppercase letters (0–2 points)

- Must contain at least **1 uppercase** letter (`A`–`Z`) to meet requirements
- Scoring:
  - 0 uppercase → **0 points**
  - 1 uppercase → **1 point**
  - 2+ uppercase → **2 points**

#### Special characters (0–2 points)

Allowed specials are: `! @ # $ % &`

- Must contain at least **1** of these to meet requirements
- Scoring is based on **distinct** (different) characters:
  - 0 distinct → **0 points**
  - 1 distinct → **1 point**
  - 2+ distinct → **2 points**

> Example: `"aa!!"` has **1 distinct** special (`!`) → 1 point  
> Example: `"aa!@"` has **2 distinct** specials (`!` and `@`) → 2 points

---

### Strength rules

1. First decide if the password **meets requirements**:  
   Requirements checklist:

- not empty
- no spaces
- length between 16–32
- has at least 1 lowercase letter
- has at least 1 digit
- has at least 1 uppercase letter
- has at least 1 allowed special character

If it does **not** meet requirements → **weak**

2. If it **meets requirements**, it is at least **normal**.

3. Then use **total points** to upgrade:

- 4–5 points → **strong**
- 6 points → **very strong**
- otherwise → keep **normal**

#### 1) Reading input from HTML


---

#### 2) String length

```js
pwd.length;
```

- Returns a **number** (character count).

Example:

```js
"hello".length; // 5
"".length; // 0
```

---

#### 3) Checking if a string contains something

```js
pwd.includes(" ");
pwd.includes("A");
pwd.includes("!");
```

- Returns **true** or **false**.

Examples:

```js
"abc".includes("a"); // true
"abc".includes("z"); // false
```

---

#### 4) Finding groups of characters (counts) using `match`

You can count digits/uppercase/lowercase using `match(...)`.

```js
(pwd.match(/[0-9]/g) || [])
  .length(
    // digit count
    pwd.match(/[A-Z]/g) || []
  )
  .length(
    // uppercase count
    pwd.match(/[a-z]/g) || []
  ).length; // lowercase count
```

Why the `|| []` part?

- If there is **no match**, `match(...)` returns `null`.
- `null` has no `.length`.
- `null || []` becomes `[]`, and `[].length` is `0`.

---

#### 5) Ternary operator `?:`

Format:

```js
condition ? valueIfTrue : valueIfFalse;
```

Examples:

```js
const msg = pwd.length === 0 ? "empty" : "not empty";

const digitPoints = digitCount >= 2 ? 2 : digitCount === 1 ? 1 : 0;
```

---

#### 6) Logical operators (Week 2 essentials)

##### AND (`&&`)

All conditions must be true:

```js
const ok = len >= 16 && len <= 32;
```

##### OR (`||`)

At least one must be true:

```js
const bad = len < 16 || len > 32;
```

##### NOT (`!`)

Flips true/false:

```js
const noSpaces = !pwd.includes(" ");
```

---

#### 7) “Cap at 2” pattern (for symbol points)

If a value is greater than 2, force it to 2:

```js
const capped = value > 2 ? 2 : value;
```
