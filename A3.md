## Overview

In this assignment you will build a small “dashboard-style” website in two phases:

- **Part 1 (HTML/CSS):** You will design a clean, readable website layout with a table, buttons, a dropdown, images, and a footer. At this stage the website is mostly _visual_—it should look like a real interface even if most buttons do not fully work yet.
- **Part 2 (DOM + JSON):** You will make the interface functional using JavaScript. You will switch between two tables, generate one table from embedded JSON, and implement sorting, reset, and dark mode.

### What this teaches you

By completing this assignment, you will practice skills you will use in real websites and apps:

**Part 1 teaches:**

- Correct HTML document structure (`head`, `body`, `header`, `footer`)
- Building and formatting tables with `tr`, `th`, `td`
- Using semantic structure and consistent layout
- CSS styling (buttons, spacing, alignment, readability)
- File/folder organization (images in `/img`)

**Part 2 teaches:**

- Reading and parsing embedded JSON
- Building HTML elements using the DOM (`createElement`, `appendChild`)
- Updating the UI based on user interactions (buttons, dropdowns)
- Input validation (`prompt`, `alert`)
- Implementing common UI features: sorting, reset, theme toggling

### What’s important (what I will grade heavily)

- **Correct structure** (HTML elements used correctly and organized)
- **Professional look** (readability, spacing, alignment, consistent styling)
- **Correct DOM logic** in Part 2 (table switching, JSON table generation, sorting validation, reset behavior)
- **Dark mode readability** (colors must be adjusted so everything is easy to read)

### What’s not important (don’t over-focus)

- You do **not** need advanced animations or fancy effects.
- You do **not** need perfect graphic design.
- You do **not** need frameworks or external libraries.
- You do **not** need npm or a local server (we embed JSON inside HTML).

## Part 2 — DOM + JSON (app.js) [60 PTS]

In Part 2 you will make the website functional.

### JSON requirements

- Embed JSON in the HTML (no external file, no fetch).
- JSON must contain:
  - **5 entries minimum**
  - **4 columns minimum**

- The JSON table must relate to the original table (example: sales performance for the cars inventory).

### JavaScript requirements (app.js)

Implement:

1. **Help button**
   - Shows help instructions explaining how the website works.

2. **Second table (hidden by default)**
   - The dropdown switches between:
     - Table 1: the original HTML table
     - Table 2: a dynamically generated table from embedded JSON

   - Use `forEach()` loops to render JSON into the table body.
   - Styling must match Table 1.

3. **Sort button**
   - Uses `prompt()` to ask for a column name.
   - Only accepts column names that exist in the **active table**.
   - If invalid -> show `alert("Invalid Input")` and do nothing.
   - If valid → sort active table by that column (A–Z / numeric low–high).

4. **Reset button**
   - Resets the active table back to default sorting **by the first column** (ascending).

5. **Dark Mode button**
   - Clicking `Dark Mode`:
     - changes text to `Light Mode`
     - switches to dark theme:
       - background darker
       - text white
       - buttons become lighter versions
       - table colors adjusted for readability

   - Clicking `Light Mode` restores normal theme.

### Advanced Search & Highlighting [6 PTS]

Add a new button called **`Advanced Search`**. This feature must work on **both tables**, but the rules differ slightly for each table.

When **Table 1** is active, clicking **Advanced Search** must:

1. Ask the user for a search expression using `prompt()`
2. Find all records (rows) matching the expression
3. **Highlight** matching rows (e.g., change background + outline)
4. Add a **star icon (★)** to a dedicated column or at the start of the row

#### Required input formats (Table 1)

Your Table 1 contains a mix of **text columns** (like make/model/color) and **numeric columns** (like year/price/quantity). Your search must support:

**✅ Text search (exact match, case-insensitive)**

User input examples:

- `Toyota`
- `Camry`
- `blue`

Rules:

- If the input does **not** contain an operator (`> < =`), treat it as a text query.
- Match against **any text column** in Table 1 (or at least the key text columns you define).
- Must be **case-insensitive**.

Expected behavior:

- Input: `toyota` → highlight all Toyota rows
- Input: `BLUE` → highlight all rows where color is blue

**✅ Numeric comparisons (must support `>`, `<`, `=`)**

User input examples:

- `year > 2019`
- `price < 20000`
- `quantity = 4`

Rules:

- Format must be:
  **`column operator value`**
  (spaces optional, but recommended)
- Valid operators: `>`, `<`, `=`
- Column names must match columns in the active table (case-insensitive).
- If user enters an invalid column/operator/value → show `alert("Invalid Input")` and do nothing.

Expected behavior:

- `year > 2020` highlights only rows with year 2021, 2022, etc.
- `price < 25000` highlights rows under 25000
- `quantity = 2` highlights only rows with quantity equal to 2

### Star requirement

- You must add a visible **★** marker for highlighted rows.
- Recommended: add a new first column header called `★` and place the star there.

### Reset interaction

- Clicking `Reset` must remove all highlights and stars and restore the table.
